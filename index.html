TP PCA / PRA — Infrastructure Active Directory
Active Directory • DNS • DHCP • Plan de Continuité & Reprise d'Activité
Windows Server Active Directory 📄 Ouvrir le document
Ce TP consiste à mettre en place une infrastructure réseau complète avec un contrôleur de domaine Active Directory, DNS et DHCP sur Windows Server virtualisé, tout en appliquant les notions de PCA (Plan de Continuité d'Activité) et PRA (Plan de Reprise d'Activité).
ÉTAPE 1
Infrastructure & VM
Création de la machine virtuelle SRV-AD sous Windows Server sur VirtualBox. Réseau interne configuré, IP statique attribuée : 172.16.1.10.
ÉTAPE 2
Installation Active Directory
Installation du rôle AD Domain Services via PowerShell. Création du domaine btssio.local et promotion du serveur en contrôleur de domaine.
ÉTAPE 3
Configuration DNS & DHCP
Installation du rôle DHCP. Création de l'étendue d'adresses IP 172.16.1.50 → 172.16.1.150 avec gateway et DNS configurés.
ÉTAPE 4
Utilisateurs & Tests
Création de l'utilisateur Jean Dupont, vérification PowerShell, connexion d'un client au domaine avec attribution d'IP automatique.
PCA / PRA
Plan de Continuité & Reprise
PCA : DC secondaire, règle DHCP 80/20, sauvegardes Windows Server Backup.
PRA : Restauration System State, bascule sur serveur de secours — RTO < 4h / RPO < 24h.
Limite : Pas de redondance matérielle dans ce TP.
✅ Infrastructure complète opérationnelle — Active Directory, DNS, DHCP fonctionnels — Client connecté au domaine avec succès

Architecture réseau sécurisée à trois zones (WAN, DMZ, LAN) déployée avec Docker et iptables. Objectif : contrôler strictement les flux entre les zones et simuler un environnement d'entreprise avec pare-feu et NAT.
ÉTAPE 1
Création des 3 réseaux Docker
docker network create --subnet 172.30.0.0/24 wan_net
docker network create --subnet 172.20.0.0/24 dmz_net
docker network create --subnet 172.10.0.0/24 lan_net
ÉTAPE 2
Pare-feu iptables — Politique DROP + règles HTTP
iptables -P FORWARD DROP
iptables -A FORWARD -i eth0 -o eth1 -p tcp --dport 80 -j ACCEPT
iptables -A FORWARD -i eth2 -o eth1 -p tcp --dport 80 -j ACCEPT
iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
ÉTAPE 3
Serveur Web Nginx en DMZ
Déploiement Nginx dans la DMZ avec routes statiques vers le LAN (172.10.0.0/24) et le WAN (172.30.0.0/24) via le pare-feu.
ÉTAPE 4
Clients LAN & WAN — Tests de validation
docker exec -it lan_client curl http://172.20.0.3
docker exec -it wan_client curl http://172.20.0.3
✅ Architecture validée — Flux HTTP LAN→DMZ et WAN→DMZ fonctionnels — Isolation réseau effective

Mémoire BTS SIO — Option SISR
Sécurisation WordPress monchauffeurspl.fr & suivi de carburant — 2024/2025
WordPress Excel / VBA 📄 Ouvrir le document
Lors de mon alternance chez Grand Transport Ouvert, deux missions principales : sécuriser le site WordPress de l'entreprise (spam massif, failles de sécurité) et créer un outil de suivi automatisé de la consommation de carburant de la flotte de véhicules.
MISSION 1
Sécurisation du site WordPress monchauffeurspl.fr
Problèmes : Absence SSL, plugins obsolètes, 20+ spams Bitcoin/crypto, aucune sauvegarde.

Solutions déployées :
— Mise à jour complète WordPress et plugins
— Antispam Bee : filtre RGPD sans Captcha
— WP Armour : technique Honeypot, champ invisible pour piéger les bots
— Liste noire de mots-clés (btc, crypto, viagra...) et bannissement d'IP malveillantes
— Stratégie de sauvegarde SQL régulière

Résultat : élimination quasi-totale du spam.
MISSION 2
Outil de suivi de carburant — Excel & VBA
Problème : Aucun suivi de consommation, risque de vol non détectable.

Réalisation :
— Import données AS24 (CSV/Excel), nettoyage des doublons et formats
— Formules SI / RECHERCHEV / INDEX-EQUIV, calcul automatique litres/km
— Mise en forme conditionnelle pour détecter les anomalies
— Tableaux croisés dynamiques (TCD) par véhicule et par période
— Formulaire VBA pour saisie automatisée et génération de rapports

Résultat : gain de temps, réduction des erreurs, meilleure visibilité sur les dépenses.
BILAN
Compétences acquises & Perspectives
Compétences : Cybersécurité WordPress, analyse de vulnérabilités, automatisation Excel/VBA.
Perspectives : Activation HTTPS (SSL OVH), migration PHP 8.3, monitoring temps réel.
✅ Site sécurisé validé — Outil carburant validé par l'entreprise — Compétences cybersécurité et automatisation acquises
